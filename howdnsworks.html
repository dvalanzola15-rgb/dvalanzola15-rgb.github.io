Steps in a DNS Lookup

  1. A user enters a domain name into a browser.
  2. The browser and operating system check their caches for a recent answer.
  3. If no answer is cached, a recursive resolver begins the lookup.
  4. The resolver asks a root server where to find the correct top-level domain server.
  5. The top-level domain server directs the resolver to the authoritative name server.
  6. The authoritative server returns the requested DNS record.
  7. The resolver gives the IP address to the browser, which then connects to the website.

Caching and Performance

DNS caching reduces repeated work. A resolver can store an answer for the amount of time defined by the record's Time to Live value. Caching improves speed and decreases traffic, but outdated cached information can remain until the Time to Live expires.
